Regenerate `lightspec/SPEC.md` — a derived, read-only digest of all requirements.

The digest is **never authoritative**: it is rebuilt from source each run, so it cannot drift. It does not merge, apply, or reconcile deltas — it concatenates requirements, attributed to their source change.

---

**Steps**

1. **Collect specs** — gather every `spec.md` under `lightspec/changes/` and `lightspec/archive/`.

2. **Empty case** — if no `spec.md` exists in either location, output `No specs to digest` and stop. Do not write or overwrite `lightspec/SPEC.md`.

3. **Extract requirements** — from each spec, pull every `### Requirement:` block from its `## Deltas` section (mirror `ls-list`'s parsing). Keep each requirement's heading and its scenarios.

4. **Write `lightspec/SPEC.md`**, overwriting any prior copy. Start with a generated-header, then list requirements grouped by source change:

   ```
   <!-- GENERATED by /ls-spec — do not edit. Rebuilt from lightspec/changes/ + archive/. -->

   # SPEC — derived digest

   ## <change-name>  (active | archived)

   ### Requirement: <text>
   <scenarios…>
   ```

5. **Report** — one line: how many requirements from how many changes were written.

---

**Rules**
- Read each `spec.md` once — no re-reads
- Never modify any source `spec.md`; never touch the archive step
- Order: active changes first, then archived; stable (alphabetical) within each group
- The digest is regenerable output — always overwrite, never merge with the existing SPEC.md
